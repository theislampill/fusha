# Qamus current dataset — complete public-safe baseline

This directory is the **complete, queryable, redistribution-safe** export of all **2,092**
current Qamus entries from the live Qamus store (`qamus.dawah.wiki`). It is the canonical
offline copy the Fusha engine (sarf/nahw skills, source-address graph, corpus→Qamus pipeline)
reads — **no live server access required**.

> Generated by `tools/export_current_qamus_dataset.py` against the authoritative live entry
> JSON store. The exporter **never writes to the live site**; it only reads entry JSON and
> emits sanitized output. Reproducible: same store → byte-identical output.

## Files

| file | what |
|---|---|
| `entries.jsonl` | one public-safe entry per line (full content), id-sorted |
| `entries.min.jsonl` | compact: `id, headword, root, section, translit, gloss, n_examples` |
| `entry-manifest.json` | counts, schema version, generation params, dropped-field log |
| `source-keys.json` | source_key → entry-id map + uniqueness report |
| `checksums.json` | sha256 + byte size of every emitted file |
| `export-report.md` | human summary of this export |
| `../../schemas/qamus-entry-public.schema.json` | the public entry contract |
| `../../indexes/current/*.json` | 7 lookup indexes (see below) |

## Public-safe contract (enforced at export, re-checked by the validator)

**Kept** (redistributable lexical content): `id, source_keys, root, root_translit, headword,
translit, section, category, definition, meaning, total_uses, tags, notes,
senses[{n,ar,translit,gloss,count}], usage[{sense,forms,examples[{ar,en,ref}]}]`.

**Stripped** (never exported): `author_uid, author_name, last_edited_by, actor_type,
created_at, updated_at, edit_count, version, versions, status, visibility, image, source`,
and any unknown field (forward-safe drop, logged in the manifest). The export contains
**no** filesystem/server paths, emails, sessions, moderation internals, raw OCR/crops, or
private provenance. `source_keys` are dictionary-source scope tokens (e.g. `v003`), not paths.

All 2,092 entries are `status=reviewed` / `visibility=public` (recorded once in the manifest
`scope`, not per entry).

## Indexes (`qamus/indexes/current/`)

`by-entry-id` (compact record per entry) · `by-source-key` · `by-root` · `by-lemma` ·
`by-normalized-surface` (norm_strict of the **headword/lemma** — the same key the live hover
`fusha` pass uses, so a token whose surface equals a lemma joins directly) · `by-quran-ref`
(`S:A` → entries citing that āyah) · `by-category` (`by_category` + `by_section`).

> **Scope note:** `by-normalized-surface` keys on the **headword**, not on every inflected
> word-form. Resolving an *inflected* hover token (e.g. أَعْمَالُنَا → entry عَمَل via enclitic
> strip + singularization) is the job of the **source-address graph** (`qamus/indexes/current/
> source-address-full.json`, built by `tools/build_full_source_address_graph.py`), which links
> each hover token to its supporting entry.

## How agents use this offline

```bash
python3 tools/query_current_qamus.py --id c59a0161fac8      # one entry
python3 tools/query_current_qamus.py --root "أ م ن"         # all entries of a root
python3 tools/query_current_qamus.py --lemma آمَنَ --full   # full entry by lemma
python3 tools/query_current_qamus.py --ref 2:13             # entries citing an āyah
python3 tools/query_current_qamus.py --section particle     # browse a POS
python3 tools/query_current_qamus.py --stats               # manifest
python3 tools/validate_current_qamus_dataset.py            # acceptance gate (fail-closed)
```

- **Entry authoring / repair:** check an existing entry, its senses, forms, example āyāt.
- **Hover-gloss authoring:** join a token's lemma via `by-normalized-surface`; cite the
  supporting entry; keep the public hover `{src:"qamus",kind:"authored"}`.
- **corpus→Qamus generation:** dedupe a candidate against this dataset before proposing it
  (the corpus→Qamus pipeline reads these committed files, not the live store).

## Regeneration

```bash
QAMUS_ENTRIES_DIR=/srv/dawah-ops/hermes-workspace/qamus-service/entries \
QAMUS_EXPORT_OUT=/tmp/qamus_export \
python3 tools/export_current_qamus_dataset.py
# then sync /tmp/qamus_export/{data,indexes} -> qamus/{data,indexes}/current and re-validate.
```

The dataset is committed directly (≈5.9 MB total, well under GitHub limits) — **not** sharded
and **not** gitignored. If it ever exceeds practical limits, shard under
`qamus/data/current/shards/` and keep `entries.min.jsonl` + indexes committed.
