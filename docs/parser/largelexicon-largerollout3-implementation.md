# Largerollout3 Implementation

This document records the repo-side implementation of the largerollout3 plan
packet. It is not live Qamus progress.

## Implemented Boundaries

- No live Qamus mutation.
- No SSH write.
- No whitelist append or replacement.
- No service restart.
- No public coverage claim.
- Public/candidate artifacts keep `src=qamus`, `kind=authored`, `lang=en`.
- Raw QAC/MCP/Tafsir/Quran.com/source-photo/OCR evidence remains private.

## Source-Card Repair

The first source-card smoke case is `n993 / 2a071cd0b50e / مَلْجَأ /
pg443.jpeg / 42:47`. The source-card repair worklist is generated by
`tools/build_qamus_source_card_repair_worklist.py` from
`qamus/indexes/largelexicon/qamus-qword-denominator.source-card-repair.json`.

The worklist is a repair and owner/source confirmation surface. It does not
write the live entry and it does not fabricate hover rows.

## Transclusion And Crosswalk

The qword denominator remains one logical Project-Xanadu-style table even though
it is physically sharded. `tools/build_largelexicon_qword_crosswalk.py` creates
a sibling crosswalk status table. Rows with null canonical Qur'an/WBW locs are
explicit `source_address_crosswalk_packet` rows, not deployable hovers.

Every crosswalk row carries source dependencies so generated projections can
round-trip from entry/card/qword to crosswalk and back. A source-card correction
must regenerate or invalidate dependent qword/crosswalk/projection rows.

## Private Cache Projection

`sources/private-acquisition-cache-policy.md` defines the private acquisition
boundary. External data can inform facts internally, but source-clean projection
must strip raw evidence, source labels, copied wording, private paths, and
process prose.

## Morphology And I'rab

`fusha/morphology/data/largelexicon-affix-compatibility.json` records the
high-yield clitic/affix/suffix families that must not collapse into host-only
hovers. `qamus/examples/largelexicon/dependency-irab-decisions.sample.jsonl`
records sample dependency/i'rab decision shapes, including `وَمَا`, attached
preposition + host, and derivative/plural morphology.

## Executor Adoption

The Qamus executor must consume all-visible-qword worklists through
`largelexicon_table_reader.py` or the Fusha CLI. Selected-word closure is not
page closure. Append-ready rows are not page closure. Parser candidates are not
page closure. The executor must handle source-card packets, crosswalk packets,
replacement lanes for undersegmented existing hovers, owner packets, scholar
packets, and renderer/validator packets before claiming tranche closure.

## Claim Boundary

This remains a dependency-free candidate/parser and rollout-support substrate.
It is not yet a CAMeL/MADAMIRA/Stanza-equivalent stack, not a certified
arbitrary-text dependency parser, and not live Qamus visual closure. Stronger
claims require measured morphology coverage, crosswalk coverage,
dependency/i'rab metrics, OOV/error rates, abstention quality, and executor
readback evidence.
